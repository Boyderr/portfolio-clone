BetterBaker

A fictional baking recipe app designed to motivate users to practice and improve their baking skills, share their creations, and track their progress. Users unlock more advanced recipes as they bake and engage with the app.

Let's Get Baking!

1. Empathize & Define

The target audience for this app was primarily females between the ages of 25-40. These women would be hobbyist bakers with busy schedules who were looking for a unique experience that would motivate them to bake more and have fun in the process. Before jumping into the design process, I created a user persona and user scenario to gain a better understanding of my target user's goals, needs, and pain points.

2. Ideate

I created a flowchart for my app to visualize the user journey from start to finish. This helped me identify potential roadblocks and improve the overall user experience, resulting in a more efficient and intuitive app design.

3. Prototype

Following my user flowchart, I created the screens necessary for the user journey using Figma. The pastel design theme and cursive font I chose for this project communicated a sense of sweetness and playfulness, and represented the icing on a cake. Keeping my target user in mind, this theme conveyed feelings of relaxation, approachability, and fun.

4. User Testing

Using Maze, I created a user test to gain insight into the usability of my design. I wanted to evaluate if users would be able to perform key tasks, and if there were any areas that might be confusing or need to be more clearly defined. After analyzing the results, I gained the necessary feedback to improve my design.

Users were unclear on the functionality of the radio buttons as they looked too much like bullet points and there was a lack of contrast separating them from the background.

There was also a lack of contrast between the pastel green elements, the white text, and the light background.

Users were similarly unclear of the functionality of these radio buttons because of their bullet-like appearance and lack of contrast.

Users also had trouble navigating back to the recipe page, as the back button needed to be more prominent and labelled.

Users were unclear how they should save the recipe and upload a photo. Adding a simple, educational onboarding message was needed to clarify this key functionality.

Going against standard conventions, I used a chef's hat as the icon for the Account tab. This user test showed that a more standard account icon would be more effective for the design.

5. Revise Prototype

Based on the issues uncovered by my user testing, I modified my prototype to improve the overall user experience. I added an onboarding message to educate users on the app's purpose and functionality, increased contrast between elements to improve readability, and swapped out the chef's hat icon for a standard user icon. I also included some animated “Baker's Tips” to clarify functions that were not immediately clear to the user.

6. Reflection

I had a lot of fun developing the UX/UI for this application. As this was a school project, we were limited to one round of user testing which I thought uncovered tons of useful analytics. However, I think with further testing and iterations, I could implement even more changes to improve the overall user experience and gain further insight into the thoughts and feelings of my end users.
